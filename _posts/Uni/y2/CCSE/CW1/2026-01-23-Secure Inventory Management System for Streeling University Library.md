The objective of this project is to develop a secure and efficient Inventory Management System
(IMS) for the Streeling University Library. The system requires a distinct separation of concerns
between general users (students) and administrative staff (librarians/admins).

Key frontend requirements identified for the Minimum Viable Product (MVP) include:
- **Authentication**: A secure Login Page and a Registration Page for students, implementing session management.
- **Main Library Catalogue**: A searchable interface featuring sorting algorithms. To optimise performance and security, the system implements server-side pagination (e.g., 100 results per page) to prevent data dumping attacks.
- **User Dashboard**: A personalised page for students displaying borrowing history, active loans, and due dates.
- **Librarian Panel**: A dedicated dashboard for librarians to view recent acquisitions, with functionality to manage inventory (upload, update, delete books) and maintain supplier and category records.
- **Audit/Log Page**: A restricted view for administrators to monitor system activity and security logs. Features include filtering, log downloading, and querying capabilities to track librarian promotion/demotion and system events.
- **Role-Based Access Control (RBAC)**: Advanced permission scoping that strictly adheres to the Principle of Least Privilege, granting access only to the API endpoints and functions required by a specific user role.

This is a coursework that was submitted as part of a The Cyber Context of Software Engineering module from Year 2 as a part of the Cyber Security Degree at University of Warwick
